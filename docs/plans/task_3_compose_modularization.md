# Task 3: Compose Modularization

## Описание
Рефакторинг Docker Compose файлов для устранения дублирования через использование `include` и базового файла конфигурации.

## Цели
1.  **DRY:** Описание общей логики (сети, тома, зависимости) в одном месте.
2.  **Гибкость:** Легкое переключение между DEV и PROD через переопределения.
3.  **Читаемость:** Уменьшение размера отдельных файлов конфигурации.

## План реализации
1.  **Создание `docker-compose.base.yml`:**
    *   Определение всех сервисов (`agent`, `rag`, `user`, `web-ui`).
    *   Настройка внешних сетей (`rag_rag_network`, `web_ui_network` и т.д.).
    *   Настройка `depends_on` и `healthcheck`.
2.  **Настройка `docker-compose.dev.yml`:**
    *   Использование `include: [docker-compose.base.yml]`.
    *   Установка `build.target: dev`.
    *   Монтирование локальных директорий через `volumes`.
    *   Проброс портов для отладки.
3.  **Настройка `docker-compose.prod.yml`:**
    *   Использование `include: [docker-compose.base.yml]`.
    *   Использование готовых образов из GHCR.
    *   Настройка `restart: always`.
    *   Удаление `volumes` с исходным кодом.

## Критерии приемки
- [ ] Команда `docker compose -f docker-compose.dev.yml up` запускает все сервисы в режиме разработки.
- [ ] Команда `docker compose -f docker-compose.prod.yml up` запускает систему на базе образов без использования локального исходного кода.
- [ ] Отсутствует дублирование определений сетей и томов между файлами.
