# Task 5: Deployment Scripts & Build Automation

## Описание
Адаптация существующих bash-скриптов под новую модульную структуру Docker Compose и использование multi-stage билдов. Обеспечение удобной локальной сборки PROD-образов.

## Цели
1.  **Актуализация:** Приведение `start-dev.sh`, `start-prod.sh` и других скриптов в соответствие с новыми именами файлов и структурой Compose.
2.  **Локальная автоматизация:** Создание или обновление скриптов для ручной сборки образов с использованием конкретных `target` стадий (например, `build-prod.sh`).

## План реализации
1.  **Скрипты запуска:**
    *   Обновление `start-dev.sh`: использование `docker compose -f docker-compose.dev.yml up -d`.
    *   Обновление `start-prod.sh`: использование `docker-compose.prod.yml`.
2.  **Скрипты сборки:**
    *   Обновление `build-prod.sh`: добавление флагов `--target prod` для всех сервисов.
    *   Добавление логики тегирования образов перед пушем в GHCR (если требуется ручной пуш).
3.  **Документация:**
    *   Обновление `README.md` и `.kilocode/rules/deploy.md` с описанием нового процесса запуска и сборки.

## Критерии приемки
- [ ] Скрипт `start-dev.sh` корректно поднимает систему в режиме разработки.
- [ ] Скрипт `start-prod.sh` корректно поднимает систему в режиме продакшн.
- [ ] Скрипт `build-prod.sh` успешно собирает оптимизированные образы (stage: prod) локально.
