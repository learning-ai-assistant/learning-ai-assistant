# Task 1: Python Unified Dockerfiles

## Описание
Перевод всех Python-сервисов (`agent_service`, `rag`, `user_service`, `test_generator`) на использование единого `Dockerfile` с поддержкой multi-stage сборок и менеджера пакетов `uv`.

## Цели
1.  **Унификация:** Один `Dockerfile` для DEV и PROD сред внутри каждого сервиса.
2.  **Скорость:** Использование `uv` для кэширования зависимостей и быстрой установки.
3.  **Безопасность:** Использование non-root пользователя в PROD образах.
4.  **Размер:** Минимизация финального образа за счет исключения инструментов сборки и кэшей `uv`.

## План реализации
1.  **Base Stage:**
    *   Использование `python:3.13-slim-bookworm`.
    *   Установка `uv` фиксированной версии (0.9.26).
2.  **Builder Stage:**
    *   Копирование `pyproject.toml` и `uv.lock`.
    *   Установка зависимостей в изолированный `.venv` через `uv sync --frozen --no-dev`.
3.  **Dev Stage (Target):**
    *   Установка всех зависимостей (включая dev).
    *   Настройка окружения для hot-reload (код монтируется через volume).
4.  **Prod Stage (Target):**
    *   Копирование только `.venv` из Builder.
    *   Копирование исходного кода.
    *   Настройка `USER nonroot`.
    *   Точка входа через `python -m app`.

## Критерии приемки
- [ ] Сервис успешно собирается с `--target dev` и запускается с hot-reload.
- [ ] Сервис успешно собирается с `--target prod`, имеет минимальный размер и запускается от не-привилегированного пользователя.
- [ ] Время повторной сборки при изменении кода (не зависимостей) составляет менее 5 секунд.
