# Task 4: Runtime Configuration for Frontend

## Описание
Переход от статических переменных окружения Vite (запекаемых при сборке) к динамической конфигурации, загружаемой в рантайме. Это позволит использовать один и тот же Docker-образ для разных окружений без пересборки.

## Цели
1.  **Портативность:** Один артефакт (Docker image) для всех сред.
2.  **Гибкость:** Изменение адреса API через файл конфигурации на сервере.

## План реализации
1.  **Фронтенд:**
    *   Создание файла `public/config.json.template`.
    *   Изменение логики инициализации приложения: загрузка `config.json` перед рендерингом React.
    *   Замена `import.meta.env.VITE_API_URL` на значение из загруженного конфига.
2.  **Docker/Nginx:**
    *   В `prod` стадии Dockerfile добавить скрипт `entrypoint.sh`.
    *   Скрипт при старте контейнера читает переменные окружения и подставляет их в `config.json` (например, через `envsubst`).
    *   Nginx раздает итоговый `config.json`.

## Критерии приемки
- [ ] Приложение успешно загружает настройки из `config.json`.
- [ ] Можно изменить URL бэкенда, просто перезапустив контейнер с новой переменной окружения, без выполнения `docker build`.
- [ ] Локальная разработка продолжает работать через `.env` или дефолтные значения.
