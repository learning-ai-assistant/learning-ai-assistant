# Разбор задачи "Valid Parentheses" (Инструкция для интервьюера)

## Основная идея
Использование **стека** для отслеживания открытых скобок. Закрывающая скобка должна соответствовать последней открытой (LIFO).

## Ключевые моменты для обсуждения
1. **Выбор структуры данных**: Почему стек? (Потому что нам важен порядок вложенности).
2. **Сложность**:
   - Временная: $O(n)$, так как мы проходим по строке один раз.
   - Пространственная: $O(n)$ в худшем случае (например, строка из одних открывающих скобок).
3. **Оптимизации**:
   - Ранний выход: если длина строки нечетная, она точно невалидна.
   - Использование словаря для маппинга закрывающих скобок на открывающие.
4. **Нюансы Python**:
   - Разница между `list` и `collections.deque` (амортизированная сложность vs гарантированная).
   - Эффективность проверки `if char in mapping` (O(1) для ключей).

## Подсказки для студента
- *Уровень 1*: "Как мы можем запомнить, какие скобки мы уже открыли, чтобы проверить их позже?"
- *Уровень 2*: "Что если использовать структуру данных, которая выдает последний добавленный элемент?"
- *Уровень 3*: "Попробуй использовать словарь, чтобы не писать много `if/elif` для каждого типа скобок."

## Пример идеального кода
```python
class Solution:
    def isValid(self, s: str) -> bool:
        if len(s) % 2 != 0: return False
        stack = []
        mapping = {")": "(", "}": "{", "]": "["}
        for char in s:
            if char in mapping:
                top_element = stack.pop() if stack else '#'
                if mapping[char] != top_element:
                    return False
            else:
                stack.append(char)
        return not stack
